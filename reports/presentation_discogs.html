Contexte & objectifs

Discogs — plus grande BDD de musique physique & numérique au monde. 1 ligne = 1 pressage individuel (vinyl, CD, fichier…) avec ses métadonnées complètes.
Dataset
  • Source : dump XML public Discogs (01/02/2026)
  • Volume : ~17 millions de releases, 45 colonnes
  • Granularité : 1 ligne = 1 pressage individuel
  • Parsing : streaming XML → CSV (~17M lignes)
Cas d'usage : aider un label à anticiper le format dominant d'une sortie et détecter l'origine géographique d'une release peu documentée.

Source de données & extraction

Pourquoi le dump XML plutôt que l'API ?
  • Couverture exhaustive (~17M releases vs limites de débit API)
  • Démarche reproductible : même snapshot = mêmes résultats
  • Parsing en mode streaming sans tout charger en mémoire
  • Export en CSV plat compatible pandas
Pipeline : XML.gz → parsing streaming → normalisation (séparateur |) → CSV (17M lignes, 45 col)

Cadrage méthodologique & pipeline

Pourquoi la classification supervisée ?
  • Les cibles prennent des valeurs discrètes & finies (5 ou 6 classes)
  • On cherche à affecter une catégorie, pas prédire un chiffre continu
  • Métrique : Balanced Accuracy — compense le déséquilibre des classes
Protocole
  • Split stratifié 80 / 20 (random_state=42)
  • Échantillon de 1M releases pour l'entraînement
  • Baseline → Logistic Regression → Random Forest
Pipeline sklearn : ColumnTransformer (OHE + imputation médiane) encapsulé dans un Pipeline — pas de data leakage, reproductible.

Interprétations clés
  • vinyl_size + has_vinyl_size = ~23 % → la taille physique est le signal le plus discriminant
  • release_type=LP → Vinyl ou CD quasi certain
  • vinyl_size=Unknown + release_type=Unknown → souvent Digital
  • genres=Electronic → fort signal Vinyl (marché historique)
  • track_count joue : EP (<6 pistes) vs LP (≥8 pistes)
Exemple :
LP + Electronic + Deep House + UK → Vinyl
Unknown + Electronic + pas de taille physique → Digital

Hypothèse : les blocs culturels (Angle B) sont plus séparables — ils partagent des traits musicaux cohérents. Le signal géo est structurellement plus faible que le signal de format.
Features & signaux géographiques attendus
  • Mêmes features qu'Axe 1, sauf country (c'est la cible)
  • format_category + decade ajoutés (signal temporel fort)
Exemples concrets
  • Electronic + 1990s + Vinyl → UK / Anglophone
  • J-Pop + CD + 2000s → Asie
  • Schlager + 1970s → Germanique
  • CD + 1990s → Japan (boom CD 90s)

Limites & points de vigilance

Axe 1 — Format
  • Snapshot catalogue : photo à une date, pas d'évolution en temps réel
  • Déséquilibre de classes : Vinyl ~47 % vs Video ~1 % → classes minoritaires mal prédites
  • Signal quasi-circulaire : vinyl_size / release_type extraits du même champ source que la cible
  • Simplification multi-label : seul le 1er style est utilisé
  • Format "Autre" exclu : Box Set, All Media (<0.3 % du catalogue)
Axe 2 — Géographie
  • Signal faible par construction : inférer l'origine sans country est une contrainte forte
  • Blocs subjectifs : Belgique, Suisse… à la frontière de deux blocs
  • Pays rares exclus : Angle A exclut ~26 % du catalogue
  • Classe "International" hétérogène : Europe, Benelux, Worldwide → fourre-tout
  • Doublons de master : une œuvre = potentiellement des dizaines de lignes
Ces limites sont connues et documentées. Elles bornent la portée des résultats et ouvrent des axes d'amélioration concrets.
9 / 10

Prochaines étapes

Modélisation
  • Comparer avec XGBoost / LightGBM pour les classes minoritaires
  • Encodage multi-label pour genres et styles (TF-IDF)
  • Ajouter label_names comme feature (certains labels = format exclusif)
Pipeline & données
  • Validation temporelle : entraîner sur <2020, tester sur ≥2020
  • Intégrer les données communautaires Discogs (community_want)
  • Déduplication par master pour réduire les biais de sur-représentation
Takeaway : les métadonnées musicales de Discogs portent un signal prédictif clair sur le format de distribution (Balanced Acc. 0.72 vs 0.20 baseline). La prédiction géographique reste un défi ouvert et stimulant.
